Prinsip Kerja Pemrosesan Query

Query adalah kemampuan untuk menampilkan data dari database untuk diolah lebih lanjut yang biasanya diambil dari tabel tabel dalam database.

Query Processing

Proses pengambilan data dari database dalam memenuhi permintaan pemakai

Parsing and Translation

Melakukan check syntax&grammar, validasi relasi yang digunakan. Mengubah query/SQL ke dalam bentuk yang bisa diolah oleh Query Processing Engine dalam bentuk aljabar relasi, struktur (tree, graph)

Optimization

Membuat evolution plan yang optimal (dengan cost yang rendah)

Evaluation

Pemilihan query-evaluation plan terbaik(cost), melakukan eksekusi terhadap query tersebut yang menghasilkan jawaban query.

Bagaiamana Cara Mengukur Biaya Queri?

Biaya evaluasi query dapat diukur dari banyaknya sumber daya sistem yang digunakan. Contohnya, pengaksesan disk,waktu CPU,dan waktu komunikasi Biasanya akses disk ini yang menjadi biaya utama, dan juga relatif mudah untuk diperkirakan. Oleh karena itu jumlah transfer blok dari disk digunakan sebagai ukuran biaya evaluasi yang sebenarnya.

Bagaimana Sebuah Sistem Database Mengevaluasi Ekspresi?

  • Materialisasi


    query-query akan dipisahkan/dibagi dulu menjadi query individual. Kemudian, akan dievaluasi querynya dan disimpan di tabel sementara pada memori. Data dari tabel sementara ini, akan digunakan untuk mengevaluasi query selanjutnya.

  • Pipeline


    Sebuah query akan dievaluasi terlebih dahulu kemudain hasilnya akan digunakan untuk query berikutnya dan begitu seterusnya dimana hasil query sebelumnya dilanjutkan ke query berikutnya.
